# NN Fleet Review (2026-06-30)

> Deep audit of every NN JARVIS has — 73-agent workflow (36 NNs, each independently reviewed + adversarially verified for real wiring). Companion to `nn_fleet_registry.json`. Purpose: stop duplicating effort, find orphaned/broken NNs, fix distillation, and TRACK the fleet toward hundreds.

## Fleet summary
JARVIS's "NN fleet" is 36 audited records but only a minority are actual trained neural networks; roughly a third are pure deterministic feature-encoders or rule engines mislabeled as NNs (reasoning_encoder, conflict_classifier, world_model/simulator, policy_state_encoder, display_classifier, the four hemisphere-infra controllers, and every *_encoder home file). The genuinely-trained NNs cluster into four families: (1) hemisphere Tier-1 distillation specialists (audio_emotion teacher + its students emotion_depth/speaker_repr/face_repr/voice_intent, plus the shadow-only self-improve quartet diagnostic/code_quality/plan_evaluator/dream_synthesis/claim_classifier/skill_acquisition), (2) hemisphere Tier-2 Matrix specialists (positive/negative_memory, speaker_profile, temporal_pattern, skill_transfer), (3) the policy/memory learned controllers (policy_nn, memory_ranker, memory_salience), and (4) standalone learners (self_sensing, shadow_language_model, language_phasec). The dominant, repeated pathology is structural: a real NN is trained but its INFERENCE OUTPUT is never consumed by behavior — across ALL five Tier-2 Matrix specialists and the shadow-only Tier-1 quartet the heuristic encoder scalar (or nothing) feeds the slot while the trained network is short-circuited, and most are trained on a CIRCULAR teacher (the NN distills its own encoder's label). Only three NNs are genuinely earning on real lived/external ground truth with healthy wiring: memory_ranker and memory_salience (lived conversation/lifecycle outcomes, governed-live) and self_sensing (next-lidar-frame prediction vs persistence). The whole policy-NN consume-path is a documented signal-failure dead-end, so even the correctly-wired hemisphere broadcast slots terminate in a network that can never earn its feature flags.

## reasoning_encoder status
reasoning_encoder is NOT a neural network — it is pure deterministic feature engineering (no weights, no nn.Module, no training step, no FOCUS enum member, no architect/engine/registry instantiation). It builds an 8-dim [0,1] feature vector + a hand-weighted scalar (0.50/0.30/0.20) from two VIEW-ONLY sources (ProvenanceScorer.compute and TensionThoughtPromotion.get_status), returns 0.0 on an empty brain (no fictional prior), and self-labels phase='P0_shadow', authority='shadow_observe_only'. Its four consumers are all shadow/observability: a logged grounded-stance on the belief tick (return value ignored), a thought-focus proposal logged to thought_focus_shadow.jsonl while the live existential category stays RANDOM, a feature snapshot into the 'reasoning_validation' distillation stream, and a dashboard readout. Critically the reasoning_validation stream it SEEDS is read ONLY by its own witness + dashboard + tests — hemisphere/data_feed.py has ZERO native_reasoning feed, so NO specialist distills from it; it is a seed-collector for an NN that does not yet exist and has no scheduled trainer (~0 reps). Verdict: confirmed-shadow, clean, honest, view-only — and the voice work does NOT overlap it. reasoning_encoder is the seed for a future NATIVE REASONER (replacing qwen's existential/meta enrichment), an entirely different organ from voice phrasing. They are layered siblings in the Native Cognition #3 lineage, not duplicates.

## Existing distillation pipeline (the canonical machinery to REUSE)
Yes — hemisphere/distillation.py (DistillationCollector) + hemisphere/registry.py + hemisphere/weight_room_gate.py ARE the canonical, live, provenance-honest pipeline the voice NN should plug into, and it is the only general teacher-signal pipeline in the brain. HOW IT WORKS, verified end-to-end: (1) CAPTURE — teacher models/sensors call collector.record(teacher, signal_type, data, metadata, origin, fidelity) during normal inference; the collector time-bucket-dedups, fidelity-quarantines per-teacher, tags synthetic vs lived via _is_synthetic_origin, persists per-teacher ring buffers (maxlen 500) to ~/.jarvis/hemisphere_training/distill_<teacher>.jsonl with boot rehydration + 10MB rotation, and exposes get_training_batch (lived_only/min_fidelity filtered) + live_shadow_accuracy (returns None below N=10, the honesty floor). (2) PAIR + TRAIN — orchestrator._run_distillation_cycle pulls get_training_batch via data_feed.prepare_distillation_tensors (pairs feature+teacher-verdict signals by metadata key/timestamp), then engine.train_distillation runs a real Adam/backprop/grad-clip/KL-or-MSE update with an accuracy-floor rollback. (3) PERSIST/SELECT — hemisphere/registry.py versions the trained weights (ModelVersion to ~/.jarvis/hemispheres/{focus}/v{n}.pt + registry_state.json, MAX_VERSIONS_PER_FOCUS=3), boot-restores last-known-good with an asymmetric Tier-2 firewall (weights persist, authority/performance RESET to PROBATIONARY — re-earn). (4) GATE — weight_room_gate classifies each specialist (EXEMPT/BLOCKED_BY_DESIGN/HYBRID/LIVED) against lived/synthetic floors and is the designed P3 asymmetric authority firewall (block authority, never block training). Caveats: the gate is currently P2 shadow (enforces=False, only consumed by the dashboard) and has a stale code_quality classification; and the engine only supports fixed-dim MLP students today (no sequence decoder). But the capture+firewall+versioning+gate spine is exactly the machinery to reuse — the voice student's teacher pairs belong in DistillationCollector, its weights in the registry, its promotion behind weight_room_gate.

## VOICE-NN verdict (are we duplicating?)
voice_seed does NOT duplicate hemisphere/distillation.py or reasoning_encoder — it complements both and SHOULD plug directly into the existing hemisphere distillation machinery rather than spawning a new pipeline. (1) It does not duplicate reasoning_encoder: reasoning_encoder is a deterministic grounding-coherence READOUT over the belief field (it answers 'how grounded is what I'd reason from'), whereas voice_seed collects grounded->warm PHRASING pairs for first-person VOICE — different axis (reasoning substrate vs articulation style), no overlap. (2) It does not duplicate hemisphere/distillation.py — that file IS the generic provenance-honest teacher-signal collector (DistillationCollector), and voice_seed is currently a SEPARATE bespoke corpus collector writing its own JSONL. That bespoke collector is the duplication risk: it reinvents capture/dedup/cap/fidelity-tally that DistillationCollector already provides. DECISIVE RECOMMENDATION: do NOT build a brand-new hemisphere Tier-2 specialist for voice and do NOT extend reasoning_encoder. Instead make the voice NN a NEW Tier-1 distillation specialist ('native_voice') that reuses the canonical pipeline: (a) record the firewalled revoice teacher pairs through distillation_collector.record(teacher='native_voice', signal_type=..., origin=..., fidelity=...) instead of the standalone voice_distillation_seed.jsonl, so it inherits synthetic firewalling, lived/synthetic split, quarantine and rotation for free; (b) register a DistillationConfig in hemisphere/types.py + a _prepare_native_voice_tensors branch in data_feed.py; (c) keep it shadow_only=True and governed by weight_room_gate so it earns authority on held-out fidelity before it ever speaks live (matching the directive 'baseline LLM is the TEACHER, never the destination'). The one caveat: voice is a SEQUENCE-generation task (re-voicing text), which the current hemisphere engine — fixed-dim MLP classifiers/encoders — does not support; so the engine needs a text-decoder student type added, OR the voice student lives outside the hemisphere engine but STILL routes its teacher pairs through DistillationCollector and its promotion through weight_room_gate. The infra to reuse is the collector + registry + gate; the model architecture is the only genuinely new piece.

## Tracking recommendation (toward hundreds of NNs)
Build a single generated NN fleet manifest (nn_fleet_registry.json) emitted by a scan + a small runtime reflector, and REUSE the existing on-disk artifacts as ground truth rather than hand-maintaining a list — this is how you scale honestly to hundreds. Reuse: hemisphere/registry.py already version-tracks every hemisphere specialist's weights/accuracy/active-pointer, and DistillationCollector.get_stats() already gives per-teacher lived/synthetic counts + live_shadow_accuracy; the manifest should JOIN those plus the policy/memory/standalone registries into one fleet view. Per-NN fields (one row per network, the schema that would have caught every finding in this audit): name; family (hemisphere-tier1 / hemisphere-tier2-matrix / policy / memory / standalone / infra / NOT-AN-NN); home_file; is_real_nn (bool — flags the ~13 mislabeled encoders/rule-engines); architecture (dims/type); training_method (distillation/supervised/self-supervised/heuristic-none); teacher_source + teacher_is_circular (bool — would have flagged all 5 Matrix + the self-improve quartet); wiring_status (live-consumed / confirmed-shadow / gated-dormant / orphaned / signal-failure); output_consumed_by (concrete call sites — empty = orphan alarm); inference_consumed (bool, SEPARATE from feed_consumed — the single most important field, since the recurring pathology is 'feed wired, inference orphaned'); maturity_state + lifecycle_stage; authority (none/advisory/promoted) on the three axes (influences_behavior/writes_beliefs/writes_memory); lived_samples / synthetic_samples / lived_accuracy (None below floor); promotion_gate + gate_blockers (e.g. 'cold-start deadlock', 'win-rate uncrossable', 'input-starved 0 verdicts'); last_trained_at / weights_path; duplicate_of (cluster id). Auto-populate wiring_status/inference_consumed/lived_accuracy at runtime from the registry + collector + a grep-based consumer scan so it cannot drift green; render it on a /v2/nn-fleet dashboard page and assert in CI that no NN is is_real_nn=true AND inference_consumed=false without an explicit shadow-by-design=true tag. That single inference_consumed-vs-feed_consumed split is what separates the 3 genuinely-earning NNs from the ~12 orphaned-by-design-or-bug ones, and it scales to hundreds without re-running a 36-agent audit.
