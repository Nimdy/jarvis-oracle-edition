## Known Issues (Inherited)

### General-Knowledge Route Over-Caution

General public-knowledge questions ("what is the Oracle in Halo") get over-constrained
responses. The route stays on NONE, fast model gets minimal context, and Ollama produces
overly cautious "I don't have data" answers. This is a routing/prompt/model-selection
issue, not a Phase B or world-model issue.

### Question Fire Reliability (wake-front-end dominant)

Recent live logs showed periods where utterances were never transcribed because wake-word
scores remained well below threshold. During those windows, the issue is perception-front-end
(wake detect), not LLM or downstream route logic.

### PVL Matrix False-Negative (status semantics mismatch)

Matrix protocol execution can complete while PVL still reports failures if contracts are
keyed to "active jobs now" instead of "observed progress (active or completed)." Shoring
patch is now in-tree; live panel reflects it after process restart.
