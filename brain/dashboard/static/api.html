Authentication

API Key Authentication

Destructive endpoints (POST, DELETE) require an API key. Read-only GET endpoints are open.

Two ways to authenticate
# Option 1: Bearer token header Authorization: Bearer <DASHBOARD_API_KEY> # Option 2: Query parameter GET /api/endpoint?api_key=<DASHBOARD_API_KEY>

The API key is auto-generated at boot and printed to the console log. Set DASHBOARD_API_KEY env var to use a fixed key. The key is also exposed at GET /api/config for the dashboard frontend.

Consciousness & Cognition 9 endpoints

▶GET/api/consciousnessConsciousness subsystem state snapshot

Full consciousness engine state including stage, confidence, mode, phase, and subsystem metrics. Updated every 2s via cache.

▶GET/api/consciousness/evolutionOperational evolution stage + emergence detector telemetry

Evolution stage and detector counters for internally generated patterns. Emergent behavior records are operational evidence signals produced by known rules over thoughts/inquiries; they are not proof of sentience, consciousness, or digital life.

▶GET/api/consciousness/thoughtsGrounded internal-thought event log

Template-based meta-thought event records with cooldowns, fingerprint dedup, and tier budgets. This is one source of internal activity, not the whole cognitive substrate and not a free-form sentience claim.

Structured observations with DeltaEffects and awareness tracking. Stance affects memory write policy via CueGate.

▶GET/api/consciousness/existentialStructured inquiry chains, LLM-gated, token-budgeted

Ticked every 120s. Produces structured chains of existential reasoning with depth limits.

▶GET/api/consciousness/philosophicalFramework debates, position evolution

LLM-gated philosophical framework debates with position tracking.

▶GET/api/consciousness/analyticsO(1) rolling-window: confidence, reasoning, health

Real-time analytics with O(1) memory footprint. Tracks confidence trends, reasoning patterns, and system health over configurable windows.

▶GET/api/kernel/performanceTick budget, cadence, p50/p95/p99 latencies

Returns kernel loop performance metrics: budget utilization, cadence multiplier, tick count, and latency percentiles.

WebSocket — Live Dashboard 1 endpoint

▶WSws://localhost:9200/wsLive dashboard snapshot push (hash-diffed, 1s interval)

Protocol

Client connects
→
Server sends full snapshot
→
Every 1s: if hash changed, push update
Message format (Server → Client)
{ "type": "snapshot", "status": { ... }, "consciousness": { ... }, "memory": { ... }, "policy": { ... }, "health": { ... }, // ... all cached subsystem snapshots }

Client → Server: any text message acts as keepalive (content ignored). Cache is rebuilt every CACHE_INTERVAL_S=2.0 seconds.

Example (websocat)
websocat ws://localhost:9200/ws

Perception Bus (:9100) WebSocket

Pi → Brain (Sensor Node to Cognitive Engine)

Binary WebSocket on port 9100. The Pi connects as a client; the brain hosts the server.

Pi connects
→
Binary frames: raw PCM 16kHz int16 (~32KB/s)
+
Text frames: PerceptionEvent JSON

Header x-sensor-id identifies the sensor. Auto-reconnect every 3s on disconnect. Max frame size 10MB. Ping interval 60s.

Connection example
# Python client (simplified) import websockets ws = await websockets.connect( "ws://brain-ip:9100", extra_headers={"x-sensor-id": "my-sensor"} ) # Send binary audio await ws.send(pcm_bytes) # Send JSON event await ws.send(json.dumps({"source": "vision", "type": "person_detected", ...}))

Pi 5 UI Server (:8080) 7 endpoints

The Pi runs an aiohttp server on port 8080 for the particle display UI, live camera feed, and debug logs.

▶GEThttp://pi-ip:8080/Particle display HTML page (Chromium kiosk)

Serves the cyberpunk particle system UI. Normally displayed in fullscreen kiosk mode on the Pi's 7" touchscreen.

▶WSws://pi-ip:8080/wsPhase state + consciousness feed for particle display
Messages (Server → Client)
// Phase change { "type": "state", "phase": "LISTENING" } // Consciousness feed for particle behavior { "type": "consciousness", "stage": "integrative", ... }
▶GEThttp://pi-ip:8080/videoMJPEG live camera stream (~10 FPS, JPEG quality 70)

Multipart MJPEG stream. Embed in <img src="..."> for live video. Content-Type: multipart/x-mixed-replace; boundary=frame.

▶GEThttp://pi-ip:8080/snapshotSingle JPEG snapshot (quality 85)

Returns a single JPEG image. Returns 503 if no camera frame is available.

▶GEThttp://pi-ip:8080/debugLive log viewer with SSE streaming

Self-contained HTML page with color-coded log levels. Uses Server-Sent Events for real-time log streaming.
