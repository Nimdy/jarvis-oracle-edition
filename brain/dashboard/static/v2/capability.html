Self-Improvement Pipeline

The closed loop: scan → propose → sandbox → human approval → merge. Stage gates whether patches can execute. Win-rates are zero until a full human-approval cycle completes — that is a cold loop, not a failure.

Codegen & Coder Model

The local Qwen coder that writes patches. infrastructure_only authority means it can propose code but never auto-merge. Standby (not running) is normal — it cold-loads on first request.

Specialist Hemisphere Learning

Per-domain neural specialists. Every one is shadow-only (live_influence=false) — telemetry, not live decisions. Accuracy here does not gate any live capability. Bootstrap accuracy of 0 is cold-start by design.

Acquisition & Plugins

How JARVIS picks up new capabilities. Scheduler in dreaming mode = low-intensity background. Plan-evaluator is a shadow NN; its accuracy is hidden until maturity ≥ meaningful (no resolved predictions yet).

Capability Deliberation

The internal think→code→validate of each plugin build, inline (no pivoting): the lane ladder, the codegen repair rounds (including what it caught and rejected), per-stage diagnostics, and the gate it's waiting on. Click a build to drill into its full plan.

Escalation & Attestation (L3 Authority)

The human-in-the-loop ledgers: L3 escalations awaiting review, and the attestation ledger of proven capabilities.

Skill Learning & Registry

Every skill JARVIS is learning or has verified. Operator Actions

Acquisition Job

Approve-plan advances to implementation; approve-deploy activates the built plugin; cancel removes the job from the active set (typed-gated).

Plugin Lifecycle

Activate brings a quarantined/disabled plugin into shadow mode; promote raises supervision tier. Disable and rollback are typed-gated.

Specialist / Skill Handoff

Approve lets JARVIS build an operational plugin/tool for a learned skill; reject sends it back. Delete forgets the skill entirely (typed DELETE gate).
